## Project Context

- Codex Keep is Benjamin's personal project and is not used by other customers or external users.
- When a useful change is complete and verified, it is OK to commit, push, and update the Sparkle release artifacts without asking for separate confirmation, unless Benjamin explicitly says not to.

## Release Notes

- Sparkle release notes must describe only the current release.
- Keep update notes short: one plain-English summary sentence and two to four bullets is usually enough.
- Do not pad notes with repeated "keeps previous behavior" bullets unless preserving that behavior is the main point of the release.
- Before publishing, read `site/Codex-Keep.md`, `site/downloads/Codex-Keep.md`, and `site/appcast.xml` together and confirm the displayed notes match the code changes in the version being shipped.
